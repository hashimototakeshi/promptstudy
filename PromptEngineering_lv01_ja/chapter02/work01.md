# 第2章 ワーク1: AIへの的確な「発注書」 - 役割、文脈、制約条件の与え方

## サンプル課題

第1章の練習問題で、曖昧なプロンプトでは良い回答が得られないことを体験しました。

**悪い例:**
```
当社の新製品のマーケティング戦略を考えてください。
```

この「悪い例」を、AIへの的確な「発注書」に変えてみましょう。以下の改善例を見て、どのような要素が加わったかを確認してください。

**良い例（発注書の形式）:**
```
# 命令書

あなたは、経験豊富なマーケティングコンサルタントです。

## 背景
当社は、健康志向の20代～40代男女をターゲットにしたスマート・ウォーターボトル「AquaMind」を新発売します。

## タスク
上記の製品の発売初期3ヶ月におけるマーケティング戦略のアイデアを3つ、具体的なアクションプランと共に提案してください。

## 制約条件
- 予算は300万円です。
- オンライン施策を主軸とします。
- 各戦略案について、期待される効果も記述してください。
```

## Point
AIへの指示は、単なる「お願い」ではありません。AIが最高のパフォーマンスを発揮できるよう、人間が**「役割」「文脈」「制約条件」**を明確に定義した「発注書」として提供することが重要です。これにより、AIの思考の方向性を定め、アウトプットの質を飛躍的に向上させることができます。

## 解説

### なぜ「発注書」形式が有効なのか？

AIは、与えられた情報（プロンプト）の中から、次に来る確率が最も高い言葉を予測して文章を生成します。指示が曖昧だと、AIはどの知識を使い、どのような立場で、どんな条件で回答すればよいか迷ってしまい、結果として一般的で当たり障りのない回答しか生成できません。

「発注書」のように構造化されたプロンプトは、AIの思考の「前提」を固める役割を果たします。

1.  **役割（ペルソナ）を与える**: `あなたは、経験豊富なマーケティングコンサルタントです。`
    - AIに特定の専門家として振る舞うよう指示することで、その役割に応じた知識や思考スタイル、言葉遣いを引き出すことができます。これにより、回答の専門性と質が向上します。

2.  **文脈（コンテキスト）を与える**: `## 背景` `## タスク`
    - AIがタスクを遂行する上で必要な前提知識や背景情報を与えます。誰が、何を、なぜ、どのようにしたいのかを伝えることで、AIとの認識のズレを防ぎ、より的確な回答を導き出します。

3.  **制約条件を与える**: `## 制約条件`
    - 回答に含めてほしい要素や、守ってほしいルール（文字数、予算、期間、トンマナなど）を明確に指定します。これにより、アウトプットの範囲を限定し、望まない回答が生成されるのを防ぎ、より要求に近い結果を得ることができます。

### サンプル課題解答

「悪い例」のプロンプトでは、AIは何の製品かも、どんな戦略を、どんな条件で考えれば良いか全く分かりません。そのため、「一般的なマーケティング戦略」として、SNS活用、コンテンツマーケティング、インフルエンサーマーケティングといった、誰にでも当てはまるような選択肢を挙げることしかできません。

一方、「良い例」のように役割、文脈、制約条件が与えられると、AIは以下のように思考を具体化します。

-   **役割**: マーケティングコンサルタントとして思考を開始する。
-   **文脈**:
    -   製品は「スマート・ウォーターボトル」。
    -   ターゲットは「健康志向の20代～40代」。
    -   目的は「発売初期3ヶ月の戦略」。
    -   これらの情報から、ターゲットに響きそうな「健康」「パフォーマンス向上」「ガジェット」といった切り口でアイデアを考えよう。
-   **制約条件**:
    -   予算は300万円。大規模なテレビCMなどは非現実的だな。
    -   オンライン施策が中心。SNS広告やインフルエンサー、Webメディアが中心になるだろう。
    -   効果も記述する必要がある。具体的なKPI（例: 売上本数、Webサイトへのアクセス数）も想定して提案しよう。

このように、前提条件が明確になることで、AIはより具体的で、実行可能性の高い、的を射た戦略を立案できるようになります。

## 練習問題

あなたは、社内の業務効率化を進めるために、AIチャットボットの導入を検討しています。上司にそのメリットを伝えるための、説得力のあるメール文を作成したいと考えています。

以下の要素を盛り込んだ「発注書」形式のプロンプトを作成し、AIにメール文案を作成させてみましょう。

-   **役割**: 経験豊富なITコンサルタント
-   **文脈**:
    -   宛先: AIに詳しくない、多忙な部長
    -   目的: 社内へのAIチャットボット導入の承認を得ること
    -   伝えたいこと: 導入による具体的なメリット（問い合わせ対応の自動化、業務時間の削減）と、懸念されそうな点（セキュリティ）への配慮
-   **制約条件**:
    -   件名は「【ご提案】業務効率化のためのAIチャットボット導入について」とする
    -   A4用紙1枚に収まる程度の簡潔な文章にする
    -   専門用語を避け、平易な言葉で説明する
